## Suggested README Positioning

- Lead with the outcome, not the template.
- Describe Obsidian as the best viewer, not the only entry point.
- Emphasize persistent Markdown knowledge rather than chat history.
- Frame the workflow as `raw sources -> maintained wiki`.

## Manual GitHub Updates

These changes need to be done in the GitHub repository settings manually:

- rename the repository slug if you want the URL to match `agent-wiki`
- update the repository description
- add the suggested topics
- optionally pin the repository on your profile with the new positioning

## Notes

The current repository slug can remain unchanged while the README and public positioning already use `Agent Wiki`. Renaming the repo later is optional, not required for the new packaging to work.
